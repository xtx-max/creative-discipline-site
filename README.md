# 十倍周期雷达

一个独立静态网站，用来做“A股一周期十倍候选”的条件筛选、路径分析、风险闸门和复盘跟踪。

当前版本内置 A 股示例股票池，覆盖沪市主板、深市主板、科创板和创业板，适合验证产品界面、筛选逻辑和十倍路径评分；真实使用前需要接入 A 股行情、财报、公告、龙虎榜/资金流、新闻、成交量和数据更新时间。

## 运行

直接打开：

```text
index.html
```

或在目录内启动任意静态服务器：

```bash
python3 -m http.server 8127
```

本地校验：

```bash
./verify.sh
```

## 上传到 GitHub Pages

这个目录是独立 Git 仓库，网站文件就在仓库根目录。推荐上传方式：

```bash
git remote add origin <你的 GitHub 仓库 URL>
git push -u origin main
```

也可以用仓库内脚本完成校验、绑定远程和推送：

```bash
./push-to-github.sh <你的 GitHub 仓库 URL>
```

推送后，在 GitHub 仓库的 `Settings -> Pages` 中选择：

- Source: `GitHub Actions`

`.github/workflows/pages.yml` 会在 `main` 分支推送后自动发布静态网站。`.nojekyll` 已放在本目录中，避免 GitHub Pages 对静态资源做 Jekyll 处理。

## 功能

- A 股市场板块、行业、周期阶段、路径倍数、十倍路径分等条件筛选
- A股一周期十倍、早期拐点、质量十倍、防守观察四类策略预设
- 候选股票按十倍路径分、路径倍数、综合分和风险排序
- 单只股票 70% 量化 + 30% 主观 + 十倍路径假设
- 风险闸门、反方观点、失效条件、数据缺口提示
- 一周期跟踪闸门和前五候选对比
- 支持粘贴 A 股 CSV 或 JSON 数组做本地数据导入
- 数据质量闸门会检查来源时间、路径假设、反方观点、风险字段和导入缺失字段
- 研究清单使用浏览器本地存储保存候选标的
- 决策日志记录每次观察、降级、剔除或复盘的理由，并支持复制 JSON
- 一键复制当前个股分析

## 边界

- 不依赖旧项目、旧后端、旧数据库或旧启动脚本。
- 不输出个股买卖建议。
- “十倍”只是研究路径假设，不代表收益承诺。
- 当前 A 股示例数据不代表实时行情、真实推荐或投资意见。
- 本地导入、研究清单和决策日志只保存在当前浏览器，不会同步到云端。
- 只用于研究筛选、分析记录和复盘。
